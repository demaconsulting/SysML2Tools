Title: Grid Quantisation — Constraint-Graph Compaction

Purpose:
  Side-by-side comparison of four blocks before and after grid quantisation (Step 3).
  Shows that snapping uses constraint-graph compaction (order-preserving, no reordering,
  no overlap) rather than independent per-block rounding.

Left panel — "Before: arbitrary positions":
  Four blocks at irregular pixel positions — not aligned to any grid.
  Block sizes vary slightly and do not match a common grid unit.
  Connector anchors land at arbitrary sub-grid coordinates.
  Faint background dots or a fine grid suggests a pixel grid that nothing aligns to.
  Visual impression: slightly off, misaligned.

Right panel — "After: G-aligned (constraint-graph compaction)":
  The same four blocks, now snapped to a visible G-unit grid (shown as faint dotted
  lines spaced G apart). Blocks are G-aligned in both position and size.
  The block order (left-to-right, top-to-bottom) is preserved — no block has jumped
  over a neighbour.
  Connector anchors now land exactly on grid intersection points.
  A dimension annotation shows "G" labelling one grid unit.
  A note: "order preserved — no overlap possible".

  Between the panels draw a right-pointing arrow labelled "Step 3 compaction".

Key message:
  Independent per-block rounding (round each coordinate separately) can reorder
  neighbours or create overlaps. Constraint-graph compaction assigns every block its
  minimum grid-aligned coordinate such that order and separations are honoured by
  construction. The result is guaranteed feasible and grid-aligned.
